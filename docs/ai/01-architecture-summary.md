● Architecture Summary

  1. 主流程（从输入到输出）

  1.1 Workflow Save/Load 链

  前端 WorkflowEditor → workflowApi.saveWorkflow()
      ↓
  api/routes/workflow_routes.py:save_workflow()
      ↓ normalize + validate
  api/workflow_normalizer.py → api/workflow_validator.py
      ↓ 通过后
  api/workflow_converter.py:editor_schema_to_yaml()
      ↓
  api/workflow_yaml_io.py:dump_canvas_workflow() → workflows/{canvas_id}/workflow.yaml

  Load 链（反向）：
  workflows/{canvas_id}/workflow.yaml
      ↓
  api/workflow_yaml_io.py:load_yaml_workflow()
      ↓
  api/workflow_converter.py:yaml_to_editor_schema()
      ↓
  api/workflow_normalizer.py:normalize_workflow_editor_data()
      ↓ 返回 WorkflowEditorData
  前端 workflowApi.loadWorkflow()

  1.2 Run 执行链

  前端 runApi.runDraft() → POST /api/workflows/{canvas_id}/run-draft
      ↓
  api/routes/run_routes.py:run_workflow_draft()
      ↓
  api/workflow_normalizer.py + api/workflow_validator.py
      ↓
  api/workflow_run_service.py:execute_draft_workflow()
      ↓
  core/engine.py:WorkflowEngine.run()
      ↓ 按拓扑序执行节点
      ├─ InputNode: 从 input_state 读取 → 发布到 current_state
      ├─ PromptNode: 解析 bound_inputs → 渲染 prompt → 调用 LLM → 发布输出
      └─ OutputNode: 聚合 bound_inputs → 发布最终输出
      ↓
  core/execution_types.py:WorkflowExecutionResult
      ↓
  api/run_result_mapper.py → 前端 RunResultPanel

  1.3 Model Resource 链

  config/model_resources.json
      ↓
  storage/model_resource_store.py:load_model_resource_record_map_or_empty()
      ↓
  core/model_resource_registry.py:load_model_resource_registry()
      ↓
  core/engine.py:resolve_model_resource() → core/llm.py:get_llm()
      ↓
  langchain_openai.ChatOpenAI.invoke()
