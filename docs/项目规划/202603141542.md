【业务场景解释】

文章扩写流程：

* 适合同一个 workflow 下创建多个独立 session
* 每篇文章一个新 session
* 不共享上下文，避免互相污染

代码审查流程：

* 适合同一个 workflow 下保留长期 session
* 在同一 session 内多次 run 继承上下文
* 逐轮补文件、补结论、继续推进

先能看

再能改真实数据和做局部测试
再做实时反馈
再扩展流程语义
